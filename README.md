# lunagalLauncher

**中文：** 用于管理本地 **llama-server**、启动翻译器与 Galgame 辅助应用，并提供鼠标映射等鼠标增强能力。  
**English:** Manages a local **llama-server**, launches translation tools and Galgame helper apps, and adds mouse enhancements such as remapping.

## 功能一览

| 模块 | 作用 |
|------|------|
| **应用管理** | 维护可执行程序列表（翻译器、辅助工具、游戏等），单独或批量启动，支持启用/禁用、编辑、删除 |
| **llama 服务** | 配置并运行本地 **llama-server**（模型路径、网络、GPU、性能参数、预设、自定义命令行） |
| **一键「Go！」** | 先尽量拉起并等待 llama HTTP 就绪，再按列表顺序启动所有「已启用」的应用（适合 LunaTranslator + 本地 API 等场景） |
| **鼠标映射** | 按规则把鼠标按键（及扩展键）映射为鼠标键或键盘组合键；支持进程过滤、优先级、多种触发方式 |
| **日志** | 查看应用内日志；打开日志文件目录 |
| **导出 / 导入设置** | 将**完整配置**备份为 JSON 或在另一台机器上恢复（含应用列表、llama、鼠标映射、窗口布局等） |

## 详细使用说明

### 应用管理

**用途：** 把常用程序（如 **LunaTranslator**、其它翻译器、补丁器、游戏等）登记到列表中，由本程序统一启动或配合「Go！」一键启动。

**操作说明：**

1. 打开左侧 **「应用管理」**。
2. 点击 **「添加应用」**，在对话框中填写：
   - **显示名称**：任意便于识别的名字；
   - **可执行文件路径**：指向 `.exe`（可通过浏览选择）。
3. 列表每一行提供：
   - **启用开关**：关闭后，该项不会参与 **「Go！」** 批量启动，一般也不建议依赖其自动行为时再启动。
   - **启动**（播放图标）：仅启动这一条目对应程序。
   - **编辑**（笔形图标）：修改名称或路径。
   - **删除**（垃圾桶图标）：从列表移除（内置项可能不可删）。
4. 图标默认从 exe 提取；若你在配置里指定了自定义图标路径，将优先使用该图标。

**注意：** 代码中对启动请求使用了 **以管理员身份运行**（`runas`）。若系统弹出 UAC，请按需确认；若不希望提升权限，需自行改动源码中的 `ProcessStartInfo` 行为。

### llama 服务

**用途：** 在本地运行 **llama-server**，为翻译器等提供兼容 OpenAI 的 HTTP API（例如探测 `/v1/models`）。界面集中管理可执行文件、GGUF 模型、监听地址、GPU 与性能选项。

**界面区块说明：**

| 区块 | 说明 |
|------|------|
| **服务状态** | 圆点与文字表示未启动 / 运行中 / 停止 / 错误 / 过渡状态；**参数信息**只读框可展示服务端输出摘要，支持复制。 |
| **启动 / 停止 / 重启** | **启动服务**：按当前表单参数拉起进程；**停止服务** / **重启服务** 仅在服务已运行时可用。 |
| **llama-server** | 选择 `llama-server.exe`（或你实际使用的服务端可执行文件）路径。 |
| **GGUF 模型** | 选择 `.gguf` 模型文件；支持多选、历史路径等列表操作（界面提供全选/清空/删除等）。 |
| **配置预设** | 将当前主机、端口、模型路径、GPU 层数、上下文等保存为**命名预设**，便于在多套模型/参数间切换。 |
| **网络配置** | **主机**（`--host`，默认多为 `127.0.0.1`）、**端口**（`--port`，默认如 `8080`）、**日志格式**（`--log-format`）。 |
| **GPU 配置** | **GPU 层数**（`-ngl`）、**选择 GPU** / **手动指定 GPU 索引**、**启用单 GPU 模式**。 |
| **性能配置** | **上下文长度**（`-c`）、**并行工作线程数**（`-np`）；可选 **Flash Attention**（`-fa`）、**禁用内存映射**（`--no-mmap`）。 |
| **自定义命令** | **追加命令**：拼在由界面生成的命令行之后；**完整自定义命令**：若填写则**覆盖**由界面生成的全部参数（高级用户）。 |

**与「Go！」的关系：**  
一键启动时，若配置中 **llama 服务已启用**（`LaunchSettings.LlamaService.Enabled` 为 `true`），且已填写 **服务端路径** 与 **模型路径**，程序会先尝试启动 llama 并**轮询 HTTP 就绪**再启动应用列表，减少翻译器抢跑连不上 API 的情况。若 llama 被禁用、或未配置完整，则会跳过自动启动 llama；若启动失败，会提示但**仍会尝试启动其它应用**。

> **说明：** 当前版本中「是否启用 llama 自动参与一键启动」由配置文件中的 `Enabled` 字段控制；界面若未提供该项开关，可在 [配置文件位置](#配置文件位置) 中编辑 `config.json`（关闭时可将 `"Enabled": false`）。

### 一键「Go！」

**用途：** 一次触发：**（可选）** 自动启动 llama 并等待就绪 → 按列表**依次**启动所有 **已启用** 的应用条目。

**操作：** 点击左侧导航栏底部的 **「Go！」**。

**行为细节：**

- 若应用列表为空，会提示先到 **应用管理** 添加应用。
- **启动模式**：由配置项 `LaunchSettings.LaunchMode` 决定。为 `"minimized"` 时，各应用以最小化方式启动；否则为正常窗口。该字段默认在配置文件中为 `"normal"`（当前界面未必提供修改入口，可直接编辑 `config.json`）。
- 每个应用启动之间约有短暂间隔，以减轻系统瞬时负载。
- 若有启动失败，会在对话框中汇总失败条目与错误信息；部分成功时也会提示。

### 鼠标映射

**用途：** 在指定进程（或全局）下，把**物理鼠标键**（左/右/中/侧键等，部分设备需用 Raw Input / HID 匹配）映射为：

- 另一个**鼠标键**，或  
- **键盘组合**（如 `Ctrl`、`Shift+Z`）。

**页面结构：**

1. **总开关**（标题旁 **开/关**）  
   关闭后，引擎不再应用任何映射规则。

2. **全局进程过滤**（卡片）  
   - 打开后可为**所有规则**先套一层进程范围。  
   - **过滤模式**：例如仅列表内进程生效（白名单），或列表内进程不生效（黑名单）。  
   - **过滤名单**：进程名（如 `game.exe`）或可执行文件完整路径；可用 **浏览…** 添加。  
   - **任务栏区域禁用** / **屏幕边缘禁用**：勾选后，光标在任务栏或屏幕边缘（约 4px）时不应用映射，避免误触系统 UI。

3. **映射规则列表**  
   - **添加规则**：新建一条规则。  
   - 每条规则可在行内配置（具体控件以界面为准），典型概念包括：
     - **物理键**：左键、右键、中键、侧键等；部分高级键位依赖 Raw Input / HID 签名。  
     - **触发方式**：单击、按住、双击、连击等；可配时间阈值、连发间隔。  
     - **行为**：触发一次，或按住期间按间隔重复。  
     - **动作**：模拟鼠标键或键盘组合文本。  
     - **进程过滤**：可再限制本条规则仅在部分 exe 上生效；**与全局进程过滤叠加**（需同时满足）。

4. **保存**  
   修改后需通过界面提供的保存逻辑写入配置（若你改动了 `config.json`，部分选项支持热重载，取决于 `MouseMapping.HotReload` 等配置）。

**提示：** 映射会影响全局输入行为，建议先用**白名单**限定到目标游戏/程序，确认无误后再扩大范围。

### 日志

**用途：** 查看本程序写入的日志，便于排查启动失败、llama 异常、映射问题等。

**操作：**

- **刷新**：重新载入日志；提示中说明：**按住 Shift 再刷新**可从文件开头完整重载（否则可能是清空显示后仅追新）。  
- **清空**：清空当前界面显示（不一定会删除磁盘上的日志文件）。  
- **打开文件夹**：打开日志目录，便于用外部编辑器查看或打包反馈。

**日志文件位置：** 默认在程序工作目录下的 `Logs` 文件夹中，按日滚动，文件名形如 `lunagalLauncher-YYYYMMDD.log`（与 Serilog 配置一致）。

### 导出 / 导入设置

**用途：** 备份或迁移**整套**配置：应用列表、llama 参数、鼠标映射、窗口布局等（与内部 `config.json` 同结构的完整 JSON）。

**导出：** 导航栏底部 **「导出设置」** → 选择保存路径 → 得到 JSON 备份。

**导入：** **「导入设置」** → 确认提示（将覆盖当前所有设置）→ 选择此前通过本程序导出的文件。  
**注意：** 导入解析要求 JSON 内含 **`LaunchSettings`** 节点，避免误把「仅鼠标映射」等不完整文件当全量配置导入而清空应用列表。

导入成功后，鼠标映射会立即按新配置应用；主窗口布局等也会按配置重载（具体以程序内实现为准）。

## 常见问题与提示

1. **一键启动后翻译器连不上 llama**  
   在 **llama 服务** 页确认服务已监听、端口与翻译器配置一致；查看 **日志** 与 llama 窗口输出。一键流程会等待 HTTP 就绪，但若模型过大或机器过慢，仍可适当延长等待或先手动启动 llama 再点「Go！」。

2. **每次启动都弹出 UAC**  
   与当前使用 `runas` 提升权限有关；仅信任来源下使用，或自行修改启动逻辑。

3. **鼠标映射不生效**  
   检查总开关、规则是否启用、进程名单与模式是否把目标程序排除在外、是否落在「任务栏/边缘禁用」区域。

4. **配置丢失**  
   定期使用 **导出设置**；重要修改前复制一份 `config.json`。

## 环境要求与构建

- Windows 10（17763+）或 Windows 11  
- [.NET 10 SDK](https://dotnet.microsoft.com/download)  
- Visual Studio 2022（含 **使用 C++ 的桌面开发** 与 **Windows 应用 SDK / WinUI** 相关工作负载）或等价环境  
- 目标框架：`net10.0-windows10.0.19041.0`

**命令行构建：**

```powershell
dotnet restore lunagalLauncher.sln
dotnet build lunagalLauncher.sln -c Debug
```

典型输出路径：

```text
bin\x64\Debug\net10.0-windows10.0.19041.0\win-x64\lunagalLauncher.exe
```

也可用 Visual Studio 打开 `lunagalLauncher.sln`，平台选 **x64**，按 F5 调试。

**仓库结构（节选）：**

```text
Core/        # 启动、GPU/模型扫描、llama 服务管理
Views/       # 各功能页面
Services/    # 输入模拟、鼠标映射引擎等
Data/        # 配置模型与持久化
docs/        # 开发与删改记录类文档
```

## English summary

**lunagalLauncher** is a WinUI 3 desktop app for Windows: manage **llama-server**, launch helper apps (e.g. translators), and use **mouse remapping** with process filters. Use **Go!** to optionally start llama until HTTP is ready, then launch all enabled apps. Full settings can be **exported/imported** as JSON. Configuration lives in `%APPDATA%\lunagalLauncher\config.json`. See sections above for details.
